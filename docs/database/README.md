# Database Docs Index

โฟลเดอร์นี้แยกมุมมองฐานข้อมูลออกเป็น 4 กลุ่ม เพื่อใช้คุยให้ตรงกันระหว่างทีมก่อนลงมือปรับโค้ดจริง

## โครงสร้าง

- `current/` : ภาพที่ระบบใช้อยู่จริงตอนนี้ (as-is)
- `target/` : ภาพที่ต้องการไปให้ถึง (to-be)
- `erd/` : มุมมองความสัมพันธ์เชิงข้อมูล
- `questions/` : รายการคำถามที่ต้องเคลียร์ก่อน migrate

## แนวทางใช้งาน

1. เริ่มจาก `current/*` ก่อนเสมอ
2. เทียบกับ `target/*` เพื่อหา gap
3. ใช้ `questions/*` เพื่อเก็บสิ่งที่ยังไม่ชัดก่อน implement
4. ทำ migration เป็น phase ตามลำดับความเสี่ยงและผลกระทบ
